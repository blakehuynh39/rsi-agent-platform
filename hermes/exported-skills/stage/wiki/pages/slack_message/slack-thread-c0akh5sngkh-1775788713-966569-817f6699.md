The share page at route `/share/$id` needs *3 API calls*, and 2 of the endpoints are *not in the OpenAPI spec*:
Endpoints Used
Endpoint Auth In OpenAPI? Status
`GET /files/{fileId}` Public *No* Missing from spec
`GET /users/{userId}` Public *No* Missing from spec
`GET /v1/campaigns/{campaignId}` Public Yes Present

Data Flow
The route loader (<vscode-webview://1306qlkq4uuq7e3fde7i8or8q5sqbserfgumddtvk0e1m66171t8/apps/web/src/routes/share.$id.tsx|share.$id.tsx>) does:

```const file = await api.getFile(params.id);
const [publicUser, campaign] = await Promise.all([
  api.getPublicUser(file.user_id),
  api.getCampaign(file.campaign_id),
]);```
Required Data Shapes
1. *`GET /files/{fileId}`* — returns `FileResponse` with `user_id`, `campaign_id`, `file_url`, `score`, `points_awarded`, etc.
2. *`GET /users/{userId}`* — returns `PublicUser` with `name`, `points`, `avatar_url`, `referral_code`, `current_rank`, wallets, etc.
3. *`GET /v1/campaigns/{campaignId}`* — returns campaign details (this one is in the spec)
