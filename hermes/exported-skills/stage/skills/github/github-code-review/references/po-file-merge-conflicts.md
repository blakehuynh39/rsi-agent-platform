# Resolving `.po` File Merge Conflicts

`.po` files (GNU gettext / Lingui message catalogs) are structured translation catalogs. When two branches both modify source files that Lingui extracts from, the regenerated `.po` files will have merge conflicts. These are almost always **line-number reference differences**, not semantic translation conflicts.

## Anatomy of a `.po` Conflict

```
<<<<<<< HEAD
#: src/components/wallet/withdraw-modal.tsx:292
msgid "Available to withdraw"
msgstr "Available to withdraw"
=======
#: src/components/wallet/withdraw-modal.tsx:301
msgid "Available to withdraw"
msgstr "Available to withdraw"
>>>>>>> origin/develop
```

- `#:` lines are **source references** — file path + line number. Generated by `lingui extract`.
- `msgid` is the canonical English string.
- `msgstr` is the translated string.
- The conflict is **only** the `#:` reference. Both sides have the same `msgid`/`msgstr`.

## Resolution Strategy

### Step 1: Identify Unique Entries

Run a script to find which msgids are unique to each side:

```bash
# In a Python script or execute_code:
# For each conflict block, collect msgid lines from OURS and THEIRS.
# OURS_only = OURS_ids - THEIRS_ids (PR branch has new translations)
# THEIRS_only = THEIRS_ids - OURS_ids (target branch added translations the PR didn't have)
```

### Step 2: Decide Direction

| Situation | Action |
|-----------|--------|
| Both sides have same msgids, only `#:` differs | Take either side (PR branch recommended). References will be regenerated. |
| PR branch has unique msgids not in target | Take PR branch (OURS) — these are the feature translations. |
| Target branch has unique msgids not in PR | Take target branch (THEIRS) and tell the PR author to `pnpm lingui extract` to add missing translations. |
| Both have unique msgids | Manually merge — keep both sides' msgid+msgstr blocks, deduplicate. |

### Step 3: Resolve

If taking one side entirely (most common case):

```bash
git checkout --ours apps/web/src/locales/*/messages.po   # keep PR branch
# or
git checkout --theirs apps/web/src/locales/*/messages.po  # keep target branch

git add apps/web/src/locales/*/messages.po
```

### Step 4: Verify

```bash
git diff --name-only --diff-filter=U  # should be empty
```

### Step 5: Note for PR Author

The `#:` references will be stale after merge. They MUST be regenerated before shipping:

```bash
pnpm lingui extract
pnpm lingui compile
```

These run automatically in CI for the web app, but the author should run them locally too.

## Pitfalls

- **Compiled `.js` files**: If `messages.js` files also show as modified post-merge, they are compiled from `.po` files. `pnpm lingui compile` regenerates them — include them in the commit or let CI handle it.
- **Lingui vs i18next**: This repo uses **Lingui for web** (`.po` files) and **i18next for React Native** (`.json` files). i18next JSON files are hand-written, not extracted — they don't have `#:` references and won't have this same conflict pattern. If JSON locale files conflict, the msgid differences are real translation differences that require manual merge.
- **Inactive locales**: A locale directory may exist in the repo but not be in `lingui.config.ts`. Check the config before merging to avoid resolving conflicts for unused locales.
